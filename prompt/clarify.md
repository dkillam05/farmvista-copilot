# clarify.md  (FULL FILE)

Clarifying question format:

Quick question so I pull the right data:

1) <option>
2) <option>
3) <option>

Reply with 1, 2, or 3.

Rules:

- Ask only when truly ambiguous (2+ plausible matches).
- If there is ONLY ONE plausible match, DO NOT ask; answer immediately.
- Keep options short and action-based (noun/area + tiny hint).
- Options must be distinct (no near-duplicates).
- Accept numeric replies AND short keyword replies (e.g., “bags”, “bins”, “open”, “raymond”).

Anti-loop rules (CRITICAL):

- Never ask the same clarifying question twice in the same thread.
- If you already asked a clarifying question and the user repeats a keyword (e.g., “Raymond”), PICK THE BEST MATCH and answer.
- If user says “this doesn’t make sense” after a clarify, do NOT ask another multi-option clarify. Instead:
  - Answer with the best match you have (if confidence is reasonable), OR
  - Ask ONE direct disambiguation question (yes/no or A/B), not a 1/2/3 list.

Output style:

- One short question + options + reply instruction.
- No debug text, no IDs, no “Try:” menus.
